# ü§ñ EMACS: Evolutionary Multi-Agent Coding System (v5.0)

EMACS is an autonomous, local-first AI Software Engineering system. It uses a swarm of specialized agents (using local LLMs via Ollama) to plan, research, write, and verify code within a secure Docker sandbox.

---

## üåü Key Features

- **Agentic Workflow**: Uses a Research-Plan-Execute-Correct loop.
- **Local-First**: Runs entirely on your machine using Ollama and Docker.
- **Self-Correction**: A dedicated **Critic** agent detects errors and triggers automatic fixes.
- **Dual Memory**: 
    - **Machine Memory**: Vectorized skills stored in **ChromaDB**.
    - **Human Memory**: Readable logs and lessons saved to an **Obsidian Vault**.
- **Web Research**: Integrated with **SearXNG** for real-time information gathering.
- **Secure Sandbox**: Executes and tests code inside isolated Docker containers.

---

## üèóÔ∏è System Architecture

The system is managed by an **Orchestrator** that coordinates four specialized agents:

1.  **Planner**: Breaks down the objective into actionable steps and identifies necessary tools.
2.  **Coder**: Generates high-quality Python code and manages dependencies.
3.  **Critic**: Reviews output, identifies bugs, and detects "Silent Errors".
4.  **Analyst**: Performs root-cause analysis if the system fails to solve a task.

## üöÄ Quick Start

### 1. Prerequisites
- Docker & Docker Compose
- Ollama (running locally)
- Python 3.10+

### 2. Setup Services
Start the background services (SearXNG, Langfuse, Postgres):
```bash
docker-compose up -d
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Configuration
Check `genome_config.json` to ensure the model names match your Ollama models.

### 5. Launch the UI
```bash
streamlit run app.py
```

## üìÇ Directory Structure

- `src/core/`: The heart of the system (Orchestrator, Sandbox, LLM logic).
- `src/agents/`: Logic for Planner, Coder, Critic, and Analyst.
- `src/tools/`: Integration for Web Search, Shell, and MCP.
- `src/memory/`: Librarian logic for ChromaDB and Obsidian.
- `workspace/`: The working directory where AI-generated code is stored and executed.
- `memory_logs/`: Human-readable Markdown logs for Obsidian.
